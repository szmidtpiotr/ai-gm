Bulk Import Templates

← Admin Panel

Reference for JSON arrays used with admin bulk import. In the admin panel, open the matching Game Design sub-tab (Skills, Weapons, Loot Tables, …) and expand Bulk import (JSON array) — paste the array there (Dry run / Commit). This page is a copy-friendly reference; no login required here.

Skills

Skills are abilities characters develop during play, each linked to a core stat. They appear on character sheets and in roll resolution.

Field Type Required Notes
key text ✅ lowercase_snake_case, unique, 1–40 chars
label text ✅ Display name shown to players
linked_stat text ✅ Must be one of: STR, DEX, CON, INT, WIS, CHA, LCK
rank_ceiling number ✅ Max rank achievable (1–10), default 5
description text ❌ Shown in tooltips and roll hints

Example key values are often already loaded from seed data. Re-importing them returns 409 Conflict — use new keys, or in the admin panel enable Skip 409 on bulk import so existing rows are skipped.

Do not include sort_order in import JSON: the server assigns max(existing) + 1 for each new skill (and the next new skill gets the following integer). Optional sort_order in JSON is only if you need an explicit value.

Example JSON 
[
  {
    "key": "athletics",
    "label": "Athletics",
    "linked_stat": "STR",
    "rank_ceiling": 5,
    "description": "Physical effort: climbing, jumping, lifting, sprinting."
  },
  {
    "key": "arcana",
    "label": "Arcana",
    "linked_stat": "INT",
    "rank_ceiling": 5,
    "description": "Knowledge of spells, magical theory, and arcane phenomena."
  }
]

Weapons

Weapons available to characters during combat. Each weapon is linked to a stat for attack rolls and restricted to specific archetypes.

Field Type Required Notes
key text ✅ lowercase_snake_case, unique
label text ✅ Display name
damage_die text ✅ Dice notation: d6, d8, 2d6 etc.
linked_stat text ✅ Stat used for attack rolls (STR/DEX/INT etc.)
allowed_classes array[string] ✅ Subset of: ["warrior", "ranger", "scholar"]
description text ❌ Flavour text; default empty string
weapon_type text ❌ melee | ranged | spell (default melee)
two_handed boolean ❌ Two-handed weapon; default false
finesse boolean ❌ Finesse property; default false
range_m number ❌ Range in metres for ranged weapons; null if not applicable
weight_kg number ❌ Mass in kilograms; default 0
note text ❌ Internal design notes
is_active boolean ❌ Default true

Enemies

Enemy templates used by the GM during combat encounters. Each defines the base stats for that enemy type.

Bulk import accepts canonical fields (hp_base, ac_base, attack_bonus). For convenience, legacy aliases are mapped when the canonical field is omitted: hp → hp_base, ac → ac_base, atk_bonus → attack_bonus.

Field Type Required Notes
key text ✅ lowercase_snake_case, unique
label text ✅ Display name
hp_base number ✅ Base hit points
ac_base number ✅ Base armour class
attack_bonus number ✅ Bonus added to attack rolls
damage_die text ✅ Damage dice (d6, 2d8 etc.)
tier text ❌ weak | standard | elite | boss (default standard)
attacks_per_turn number ❌ Attacks per turn; default 1
damage_bonus number ❌ Flat damage bonus; default 0
damage_type text ❌ physical | magic | fire | poison | misc (default physical)
xp_award number ❌ XP granted on defeat; default 0
conditions_immune array[string] ❌ JSON array of condition keys this enemy ignores; default []
loot_table_key text ❌ Optional reference to a loot table key
drop_chance number ❌ 0.0–1.0 chance the loot table is rolled on defeat; default 1.0
note text ❌ Internal design notes
description text ❌ Flavour text for the GM
is_active boolean ❌ Default true
Example JSON

Conditions

Status effects that can be applied to characters or enemies. Each condition modifies stats for a defined duration.

Field Type Required Notes
key text ✅ lowercase_snake_case, unique
label text ✅ Display name
effect_json text ✅ JSON string with stat_mods and duration
description text ❌ Plain-language explanation shown to players
stackable boolean ❌ Whether multiple instances can stack; default false
auto_remove text ❌ When the engine removes the condition (e.g. end_of_encounter); null = manual only
is_active boolean ❌ Default true

effect_json structure (must be valid JSON encoded as a string): { "stat_mods": { "STR": -2 }, "duration": "3 turns" }

Example JSON 
[
  {
    "key": "poisoned",
    "label": "Poisoned",
    "effect_json": "{\"stat_mods\":{\"STR\":-2},\"duration\":\"3 turns\"}",
    "description": "Toxin weakens muscles. STR -2 for 3 turns.",
    "is_active": true
  },
  {
    "key": "blinded",
    "label": "Blinded",
    "effect_json": "{\"stat_mods\":{\"DEX\":-4},\"duration\":\"2 turns\"}",
    "description": "Cannot see. DEX -4 for 2 turns.",
    "is_active": true
  }
]

Items

Master catalogue entries for gear, consumables, quest objects, and other lootable or tradable items.

Field Type Required Notes
key text ✅ lowercase_snake_case, unique
label text ✅ Display name
item_type text ✅ weapon | armor | consumable | misc | quest (default misc)
description text ❌ Flavour text
value_gp number ❌ Base gold value, default 0
weight number ❌ Legacy encumbrance column; default 0. Prefer weight_kg for new data.
weight_kg number ❌ Mass in kilograms (spec-aligned); default 0
proficiency_classes array[string] ❌ JSON array, e.g. ["warrior"]; subset of warrior/ranger/scholar; default []
note text ❌ Internal design notes
effect_json text ❌ JSON string, same format as conditions
is_active boolean ❌ Default true

    "is_active": true
  },
  {
    "key": "iron_shield",
    "label": "Iron Shield",

label text ✅ Display name
description text ❌ Flavour text; default empty string
effect_type text ❌ heal_hp | restore_mana | remove_condition | add_condition | stat_buff | misc (default misc)
effect_dice text ❌ Dice expression e.g. 2d4; null uses flat effect_bonus only
effect_bonus number ❌ Flat bonus; default 0
effect_target text ❌ self | ally | any (default self)
weight_kg number ❌ Mass in kilograms; default 0

base_price number ❌ Price in copper pieces (or project currency); default 0
note text ❌ Internal design notes

Loot tables

Weighted drop tables: one object per table. Optional entries is an array of loot rows; each row references exactly one catalogue key (item_key, weapon_key, or consumable_key) that must already exist in the admin catalogues.

Field Type Required Notes
key text ✅ lowercase_snake_case, unique per table
label text ✅ Display name
description text ❌ Default empty string
is_active boolean ❌ Default true
entries array ❌ Loot rows; omit or [] for an empty table
entries[].item_key / weapon_key / consumable_key text ✅ per entry Exactly one must be set; must match an existing row
entries[].weight number ❌ Relative weight; default 10; must be ≥ 1
entries[].qty_min / qty_max number ❌ Defaults 1; must be ≥ 1 and qty_min ≤ qty_max

409 on table key: with Skip 409 enabled in the admin panel, the create-table step is skipped for duplicate keys, but entries are still POSTed (upsert) to the existing table.

Example JSON 
[
  {
    "key": "example_loot_table",
    "label": "Example drops",
    "description": "Replace keys with real item / weapon / consumable keys from your project.",
    "is_active": true,
    "entries": [
      { "item_key": "rope", "weight": 10, "qty_min": 1, "qty_max": 2 },
      { "consumable_key": "potion_healing_minor", "weight": 3, "qty_min": 1, "qty_max": 1 }
    ]
  }
]

Locations

Lokacje kampanii (makro i sub). Import działa dla zakładki Game Design → Locations i wspiera tworzenie nowych wpisów oraz patch duplikatów po key.

Field Type Required Notes
key text ✅ lowercase_snake_case, unique
label text ✅ Nazwa wyświetlana w panelu i narracji
location_type text ✅ macro lub sub
parent_id number|null ❌ Wymagane dla sub (ID lokalizacji makro)
description text|null ❌ Opis lokacji
rules object|string|null ❌ JSON z regułami albo tekst wolny
enemy_keys array[string] ❌ Lista kluczy wrogów możliwych w lokacji
npc_keys array[string] ❌ Placeholder pod Phase 9 (może być puste)
Example JSON 
[
  {
    "key": "city_varen",
    "label": "Miasto Varen",
    "location_type": "macro",
    "parent_id": null,
    "description": "Duże miasto handlowe na trakcie północnym.",
    "rules": { "no_rest": false, "fog_of_war": false },
    "enemy_keys": ["bandit", "thug"],
    "npc_keys": []
  },
  {
    "key": "tavern_hanged_man",
    "label": "Karczma Pod Wisielcem",
    "location_type": "sub",
    "parent_id": 1,
    "description": "Głośna karczma przy rynku.",
    "rules": "Plotki rozchodzą się szybciej niż zwykle.",
    "enemy_keys": ["drunk_brawler"],
    "npc_keys": []
  }
]
