Nadpisuje name + sheet_json dla istniejącego characters.id. Historia tur (campaign_turns) zostaje przypisana do tego samego id — nie tworzy nowej postaci. Treść sheet_json jak przy POST /campaigns/…/characters (archetype, stats, skills, background, inventory w JSON karty). Usuń bohatera w tabeli wywołuje DELETE /api/admin/characters/{id} — usuwa wiersz postaci, tury tej postaci i ekwipunek; kampania zostaje (może być bez bohatera). Na przyszłość / rzadkie przypadki.
