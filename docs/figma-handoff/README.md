# Figma Handoff Package

This package defines how design work in Figma maps to the current AI-GM codebase.

Purpose:
- Keep design and implementation aligned.
- Provide a reusable prompt/brief for Figma Make sessions.
- Standardize component naming between Figma and code.

How to use:
1. Start with `FIGMA_BRIEF.md`.
2. Use `COMPONENT_MAP.md` for naming in Figma components/variants.
3. Use `UI_SPEC.md` for realistic table/form sample data and UI states.
4. Share frame links back to development for implementation.

Maintenance:
- Update these files after major frontend/admin UX changes.
- Keep names synchronized with existing frontend files under `frontend/`.
